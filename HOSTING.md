# Запуск и хостинг ботов

## 1. Запуск/остановка на своём ПК (Windows)

**Запустить:**
- Двойной клик по `start.bat` в папке бота (`D:\botxd2\start.bat` или `D:\botxd\start.bat`).
- Откроется окно — бот работает, пока окно открыто.

**Или вручную в терминале (PowerShell/cmd):**
```
cd D:\botxd2
python bot.py
```

**Остановить:**
- Закрыть окно бота, или нажать `Ctrl + C` в окне.
- Принудительно остановить ВСЕ боты сразу: `taskkill /IM python.exe /F`

⚠️ Минус своего ПК: бот работает только пока включён компьютер и есть интернет.
Для постоянной работы 24/7 — ставь на хостинг (см. ниже).

## 2. Хостинг на сервере 24/7 (рекомендуется VPS на Linux)

### Шаг 1. Купить VPS
Подойдёт самый дешёвый VPS (1 ядро, 1 ГБ RAM — хватит на оба бота).
Хостинги: Aeza, Timeweb, Beget, Reg.ru, PQ.Hosting, Hetzner и т.п.
Бери **Ubuntu 22.04/24.04**. После оплаты дадут IP, логин (root) и пароль.

### Шаг 2. Подключиться к серверу
С Windows — через программу **PuTTY** или в PowerShell:
```
ssh root@IP_СЕРВЕРА
```
Введи пароль (при вводе он не отображается — это нормально).

### Шаг 3. Установить Python и git
```
apt update && apt install -y python3 python3-venv python3-pip git
```

### Шаг 4. Загрузить файлы бота на сервер
**Вариант А (проще) — через WinSCP:**
скачай программу WinSCP, подключись по IP/логину/паролю, перетащи папку `botxd2`
в `/root/`. Файл `.env` и `referral_bot.db` тоже копируй (это токен и база пользователей).

**Вариант Б — через git**, если зальёшь код на GitHub (но `.env` туда НЕ клади — там токен!).

### Шаг 5. Установить зависимости
```
cd /root/botxd2
python3 -m venv venv
venv/bin/pip install -r requirements.txt
```

### Шаг 6. Проверить запуск
```
venv/bin/python bot.py
```
Если в логах `Start polling` — всё ок. Останови `Ctrl + C` и настрой автозапуск.

### Шаг 7. Автозапуск через systemd (бот сам поднимается после перезагрузки и падений)
```
cp /root/botxd2/botxd2.service /etc/systemd/system/botxd2.service
systemctl daemon-reload
systemctl enable botxd2
systemctl start botxd2
```
Проверить статус и логи:
```
systemctl status botxd2
journalctl -u botxd2 -f
```

**Управление ботом на сервере:**
- Остановить:  `systemctl stop botxd2`
- Запустить:   `systemctl start botxd2`
- Перезапустить (после изменений кода): `systemctl restart botxd2`

### Для второго бота
Повтори шаги 4–7 для папки `botxd` (загрузи в `/root/botxd`, создай свой
`botxd.service` по образцу — поменяй в нём пути и Description), назови сервис `botxd`.

## 3. Альтернатива без своего сервера (PaaS, проще, но платно/лимиты)
- **Amvera** (ru), **Railway**, **Render**, **fly.io**, **PythonAnywhere**.
  Заливаешь код, прописываешь переменные окружения (как в `.env`), запускаешь `python bot.py`.
  Удобно, но БД (`referral_bot.db`) на таких платформах может сбрасываться при
  передеплое — нужен постоянный диск (volume) или внешняя БД.

---

## Важно при переносе
- Скопируй файл **`.env`** (в нём токен бота и настройки) — без него бот не запустится.
- Скопируй **`referral_bot.db`**, если хочешь сохранить текущих пользователей, балансы,
  промокоды и заявки. Не скопируешь — бот создаст пустую базу с нуля.
- На сервере должен быть один экземпляр бота на токен — нельзя запускать тот же бот
  и на ПК, и на сервере одновременно (будет конфликт getUpdates).
